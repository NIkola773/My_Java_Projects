Online Shop Application

A simple Spring Boot + H2 application demonstrating REST APIs, JPA, and security configuration

Overview

This project demonstrates a small online shop backend using Spring Boot, with the following features:

  • CRUD operations for Products entity via REST APIs
  • JPA/Hibernate integration with H2 in-memory database
  • Spring Security configuration for role-based access control
  • Open access routes for public endpoints

Application Structure

  • Security.java: Configures Spring Security. Admin routes like /adminPanel and /allUsers are restricted, while all other routes are publicly accessible.
  • Products.java: Entity class representing a product with id, product_name, and price.
  • JPArepo.java: Spring Data JPA repository for Products entity.
  • AppService.java: Service layer handling business logic for products. Includes methods to retrieve all products, filter by price, and add/delete products.
  • Controller.java: REST controller exposing endpoints like /products and /addProducts for client interaction.

Security & Users

The application defines two in-memory users:

  • user / user1 with role USER
  • admin / admin123 with roles ADMIN, USER

Routes restricted to ADMIN role include:

  • /adminPanel
  • /allUsers

All other routes, including product listing, are publicly accessible.

Available Endpoints

  • GET /products - Returns a list of all products
  • POST /addProducts - Adds a new product
  • Additional endpoints for filtering by price or IDs will be added later

Technologies Used

  • Spring Boot 3.x
  • Spring Security 7
  • Spring Data JPA + H2 database
  • REST APIs using Spring Web
  • Thymeleaf for frontend templating (future expansion)
